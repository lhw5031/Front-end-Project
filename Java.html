Language / Java

Java

Java 공식 문서 Java API

Java는 객체지향을 중심으로 설계된 범용 프로그래밍 언어입니다. 한 번 작성한 코드를 다양한 환경에서 실행할 수 있도록 JVM 위에서 동작하며, 웹 서버, 안드로이드, 기업용 시스템 개발에 널리 사용됩니다.

변수

변수는 값을 저장하는 이름 있는 공간입니다. Java에서는 변수를 만들 때 자료형을 먼저 적고, 그 뒤에 변수 이름과 값을 작성합니다. 

int age = 20;
String name = "Jin";
boolean isStudent = true;

자료형

Java의 자료형은 기본형과 참조형으로 나뉩니다. 기본형은 숫자, 문자, 논리값을 직접 저장하고, 참조형은 객체의 위치를 가리킵니다.

기본형

int, double, char, boolean

참조형

String, 배열, 클래스, 컬렉션 같은 객체 타입

연산자

연산자는 값을 계산하거나 비교할 때 사용합니다. 산술, 비교, 논리, 대입 연산자가 기본입니다. 

int total = 10 + 5;
boolean pass = total >= 10 && total < 20;

조건문

조건문은 상황에 따라 다른 코드를 실행합니다. 대표적으로 if, else if, else, switch가 있습니다. 

if (score >= 90) {
  grade = "A";
} else if (score >= 80) {
  grade = "B";
} else {
  grade = "C";
}

반복문

반복문은 같은 작업을 여러 번 실행할 때 사용합니다. 반복 횟수가 정해져 있으면 for, 조건 중심이면 while이 자주 쓰입니다. 

for (int i = 0; i < 3; i++) {
  System.out.println(i);
}

배열

배열은 같은 자료형의 값을 순서대로 저장하는 구조입니다. 길이가 고정되기 때문에 크기가 자주 바뀌는 데이터에는 컬렉션을 더 많이 사용합니다. 

int[] scores = {90, 80, 100};
System.out.println(scores[0]);

메서드

메서드는 특정 작업을 수행하는 코드 묶음입니다. 입력값인 매개변수를 받고, 결과값을 반환할 수 있습니다. 

int add(int a, int b) {
  return a + b;
}

클래스

클래스는 객체를 만들기 위한 설계도입니다. 객체가 가질 상태는 필드로, 수행할 행동은 메서드로 표현합니다. 

class User {
  String name;

  void sayHello() {
    System.out.println("Hello " + name);
  }
}

객체

객체는 클래스로부터 만들어진 실제 인스턴스입니다. new 키워드로 생성하고, 점 표기법으로 필드와 메서드에 접근합니다. 

User user = new User();
user.name = "Jin";
user.sayHello();

생성자

생성자는 객체가 만들어질 때 실행되는 특별한 메서드입니다. 클래스 이름과 같은 이름을 가지며, 주로 필드의 초기값을 설정합니다. 

class User {
  String name;

  User(String name) {
    this.name = name;
  }
}

this

this는 현재 객체 자신을 가리킵니다. 필드 이름과 매개변수 이름이 같을 때 특히 자주 사용합니다. 

this.name = name;

접근 제어자

접근 제어자는 클래스, 필드, 메서드를 어디에서 사용할 수 있는지 정합니다. 보통 필드는 private로 숨기고 필요한 메서드만 공개합니다.

제어자 접근 범위
public 어디서나 접근 가능
protected 같은 패키지 또는 상속 관계에서 접근 가능
private 같은 클래스 내부에서만 접근 가능

캡슐화

캡슐화는 데이터와 동작을 하나로 묶고, 객체 내부 상태를 직접 건드리지 못하게 보호하는 설계 방식입니다. 

class Account {
  private int balance;

  public void deposit(int amount) {
    if (amount > 0) {
      balance += amount;
    }
  }
}

상속

상속은 기존 클래스의 필드와 메서드를 물려받아 새로운 클래스를 만드는 기능입니다. Java에서는 extends 키워드를 사용합니다. 

class Animal {
  void speak() {
    System.out.println("sound");
  }
}

class Dog extends Animal {
}

오버라이딩

오버라이딩은 부모 클래스의 메서드를 자식 클래스에서 다시 정의하는 것입니다. 같은 호출이라도 자식 객체에 맞는 동작을 실행할 수 있습니다. 

class Dog extends Animal {
  @Override
  void speak() {
    System.out.println("bark");
  }
}

다형성

다형성은 같은 타입으로 바라본 객체들이 실제 객체에 따라 다른 방식으로 동작하는 특징입니다. 

Animal animal = new Dog();
animal.speak(); // bark

추상 클래스

추상 클래스는 직접 객체를 만들 수 없고, 자식 클래스가 공통 구조를 물려받도록 만드는 클래스입니다. 미완성 메서드인 추상 메서드를 가질 수 있습니다. 

abstract class Shape {
  abstract double area();
}

인터페이스

인터페이스는 클래스가 반드시 구현해야 하는 동작의 약속입니다. Java에서는 다중 구현이 가능해 역할을 분리하는 데 유용합니다. 

interface Flyable {
  void fly();
}

class Bird implements Flyable {
  public void fly() {
    System.out.println("fly");
  }
}

static

static은 객체마다 따로 존재하지 않고 클래스에 하나만 존재하는 멤버를 만들 때 사용합니다. 

class MathUtil {
  static int square(int n) {
    return n * n;
  }
}

final

final은 변경을 막는 키워드입니다. 변수에 붙이면 값 변경을, 메서드에 붙이면 오버라이딩을, 클래스에 붙이면 상속을 막습니다. 

final int MAX_COUNT = 10;

예외 처리

예외 처리는 실행 중 발생할 수 있는 오류 상황을 안전하게 다루는 방법입니다. try, catch, finally를 사용합니다. 

try {
  int result = 10 / number;
} catch (ArithmeticException e) {
  System.out.println("0으로 나눌 수 없습니다.");
}

ArrayList

ArrayList는 크기가 자동으로 늘어나는 리스트입니다. 배열보다 추가와 삭제가 편리해서 실무에서 자주 사용됩니다. 

ArrayList<String> names = new ArrayList<>();
names.add("Jin");
names.add("Mina");

HashMap

HashMap은 키와 값을 한 쌍으로 저장하는 자료구조입니다. 키를 이용해 원하는 값을 빠르게 찾을 수 있습니다. 

HashMap<String, Integer> scores = new HashMap<>();
scores.put("Java", 95);
System.out.println(scores.get("Java"));

제네릭

제네릭은 클래스나 메서드가 다룰 타입을 나중에 지정할 수 있게 해줍니다. 타입 안정성을 높이고 불필요한 형변환을 줄입니다. 

class Box<T> {
  private T value;

  void set(T value) {
    this.value = value;
  }
}

람다식

람다식은 짧은 함수를 간결하게 표현하는 문법입니다. 주로 컬렉션 처리, 이벤트 처리, 함수형 인터페이스와 함께 사용됩니다. 

names.forEach(name -> System.out.println(name));
